
# X市建投集团300亿债务危机紧急听证会网页

这是我公共经济学课程的一次网页作业。老师原本希望我们在已有网页或课程主题的基础上，利用 AI 工具做一个更有互动性的网页模块。我最后选择做一个围绕“X市建投集团300亿债务危机紧急听证会”的角色扮演网页。

这个网页主要想模拟一个现实中可能出现的地方债务危机场景：一个省会城市的城投集团债务规模很大，其中有300亿元债务在半年内到期。如果不处理，可能影响水务、供暖、轨道交通等公共服务，也可能引发城投债市场的连锁风险。但如果中央财政直接兜底，又会带来预算软约束、道德风险和纳税人负担的问题。

所以这个案例本身很适合公共经济学讨论，因为它不是简单的“救”或者“不救”，而是要讨论谁决策、谁受益、谁承担成本，以及财政风险会不会外溢。

## 项目背景

省会 X 市建投集团总债务约 1500 亿元，其中 300 亿元将在半年内集中到期。该集团承担水务、供暖、地铁、棚改等公共民生业务。

如果债务违约，可能导致 300 万市民公共服务受到影响，并引发全省城投债务连锁风险。省政府因此向中央财政部申请发行 300 亿特别再融资债，用于化解本次到期债务。

本项目通过角色扮演方式，引导学生思考：

- 中央财政是否应该兜底地方城投债务？
- 金融稳定和财政纪律如何权衡？
- 隐性担保是否应被打破？
- 全国纳税人是否应承担地方债务成本？
- 公共服务连续性与预算约束之间如何平衡？

## 我做了什么

这个网页是用 HTML、CSS 和原生 JavaScript 写的，没有使用后端、数据库或复杂框架，可以直接部署到 GitHub Pages，也可以本地打开运行。

网页里面设计了两个模式：

- 教师端
- 学生端

教师端主要用来控制课堂流程，比如推进听证会阶段、投放突发事件、随机点名、查看学生发言和投票结果。

学生端主要用来让同学选择角色、查看证据、生成发言、回应老师推送的事件，并且进行最终政策投票。

我这样设计，是因为如果只是一个普通网页，互动感会比较弱；但如果把它做成“老师推动剧情、学生用角色回应”的形式，就更像一次课堂听证会，也更适合班上同学 cosplay 不同角色。

## 角色设置

网页里一共有五个角色：

1. 省政府  
   立场偏向支持中央兜底，主要强调金融稳定和公共服务不能中断。

2. 财政部  
   立场偏向拒绝兜底，主要强调预算纪律、《预算法》和不能形成中央兜底预期。

3. 城投集团负责人  
   立场偏向请求救助，强调城投承担了很多公共服务任务，并不是完全自主举债。

4. 债券债权人  
   立场偏向刚性兑付，强调债市信心和投资者权益。

5. 普通纳税人  
   立场偏向反对兜底，强调纳税人没有参与举债决策，却可能被动承担成本。

## 课堂流程

我把听证会设计成几个阶段：

1. 开庭导入  
   先展示债务背景、债务规模和可能的危机后果。

2. 角色入席  
   学生选择自己扮演的角色。

3. 证据发放  
   学生根据角色选择证据，比如《预算法》约束、民生服务清单、城投债利差扩大、审计疑点等。

4. 首轮陈述  
   每个角色生成一段自己的开场发言。

5. 交叉质询  
   老师可以随机点名，也可以选择某个学生发言投屏展示。

6. 危机推进  
   老师可以投放突发事件，比如债市恐慌、供暖季临近、低效项目曝光、财政部划红线等。

7. 政策谈判  
   学生根据新的危机情况调整立场，提出审计、展期、资产处置、问责等配套条件。

8. 最终投票  
   学生以自己的角色身份投票，选择支持兜底、拒绝兜底，或者附条件救助。
